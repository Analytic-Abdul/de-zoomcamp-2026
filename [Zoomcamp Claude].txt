  ### General Computer Basics                                               
  *   **Operating System (OS):** This is the "brain" software of your       
  computer that manages everything, like Windows, macOS, or Linux.          
  *   **Kernel:** The secret heart of the OS. It is the part of the         
  software that talks directly to the computerâ€™s hardware [Conversation     
  History].                                                                 
  *   **File System:** This is how your computer organizes and stores your  
  files and folders so it can find them later.                              
  *   **Terminal / Command Line:** Instead of clicking icons with a mouse,  
  the terminal is a window where you type specific text commands to tell    
  the computer what to do.                                                  
  *   **Bash (and .bashrc):** Bash is a specific type of "language" used in 
  the terminal. A `.bashrc` file is like a list of settings or "house       
  rules" that your terminal follows every time you open it.                 
  *   **Localhost:** This is a "nickname" your computer uses to talk to     
  itself. When you go to "localhost" in a browser, you are looking at a     
  program running on your own machine rather than the internet.             
  *   **Ports:** Think of your computer like a giant apartment building.    
  **Ports** are like the specific apartment numbers. To talk to a specific  
  program (like a database or a website), you have to send data to the      
  right "apartment number" or port.                                         
                                                                            
  ### Coding and Data Tools                                                 
  *   **VS Code (Visual Studio Code):** A super-powered notepad for         
  programmers. It helps you write and organize code with special colors and 
  tools.                                                                    
  *   **Python:** A very popular "human-friendly" language used to give     
  computers instructions for things like data engineering.                  
  *   **Dependencies / Libraries:** These are "toolkits" other people wrote 
  that you can add to your project so you donâ€™t have to build everything    
  from scratch. For example, **Pandas** is a library for handling data      
  tables.                                                                   
  *   **Virtual Environment (Venv):** A "bubble" for your project. It lets  
  you install specific tools for one project without cluttering up your     
  whole computer or messing up other projects.                              
  *   **Database (like Postgres):** A high-tech digital filing cabinet.     
  Itâ€™s designed to hold massive amounts of information and let you find or  
  change it very quickly.                                                   
  *   **CSV vs. Parquet:** **CSV** is like a simple, plain text list of     
  data. **Parquet** is a "high-tech" version that is squished down to save  
  space and is much faster for big computers to read.                       
                                                                            
  ### Docker Specific Jargon                                                
  *   **Docker:** A tool that lets you "wrap up" an application and         
  everything it needs into a neat package so it works the same on any       
  computer.                                                                 
  *   **Image:** Think of this like a **recipe** or a **blueprint**. It is  
  a saved "snapshot" of a computer system that has everything ready to go   
  but isn't "alive" yet.                                                    
  *   **Container:** This is the "alive" version of the image. When you     
  "run" an image, it becomes a containerâ€”an isolated box where your app     
  lives.                                                                    
  *   **Isolated:** This means what happens inside a Docker container stays 
  inside. If you delete every file inside a container, your actual          
  computer (the **Host Machine**) is still perfectly safe.                  
  *   **Stateless:** Containers are "forgetful." If you save a file inside  
  a container and then turn the container off, that file disappears forever 
  unless you use a "Volume".                                                
  *   **Volume (or Mapping):** A "magic tunnel" that connects a folder on   
  your real computer to a folder inside the Docker container. This is how   
  you save data so the container doesn't "forget" it.                       
  *   **Dockerfile:** A simple text file that contains the "recipe"         
  instructions on how to build a Docker Image.                              
  *   **Docker Compose:** A manager tool. If you need three different       
  containers (like a database, a website, and a data tool) to work          
  together, Docker Compose lets you start them all at once using one file.  
  *   **Entry Point:** The very first command the container runs            
  automatically as soon as it "wakes up".

  ---
  2. What Does -u Do?

  git push -u origin main

  With -u (upstream):
  - Git remembers: "main branch â†’ origin/main"
  - Next time, just type: git push

  Without -u:
  - Git doesn't know where to push
  - Every time you'd need: git push origin main

  It's a one-time shortcut setup. You only need -u on the first push.
